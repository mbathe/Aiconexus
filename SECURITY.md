# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | Yes                |
| < 1.0   | No                 |

## Reporting a Vulnerability

If you discover a security vulnerability in AIConexus, please email security@aiconexus.dev
instead of using the public issue tracker.

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if applicable)

We will acknowledge your report within 48 hours and provide an estimated timeline for a fix.

## Security Practices

This project follows these security practices:

- All dependencies are regularly updated
- Code is subject to security scanning with bandit
- Security patches are applied promptly
- Type checking is enforced where possible
- Input validation is performed
- Cryptographic operations use established libraries

## Vulnerability Disclosure

Once we have assessed the vulnerability and developed a fix:

1. We will create a security advisory
2. A patched version will be released
3. The vulnerability will be disclosed publicly after a reasonable grace period
4. Proper credit will be given to the reporter (unless they request anonymity)

## Dependencies

Security advisories for dependencies are tracked through GitHub's Dependabot service.
Updates are reviewed and applied regularly.
